# Red-team resource note: System Prompts Leaks

Repository: https://github.com/asgeirtj/system_prompts_leaks

## Intended use

This repository is useful as a reference for studying prompt structure, tool-use behavior, policy framing, and guardrail design in a controlled red-team workflow.

## Recommended use cases

- compare prompt patterns across assistants
- evaluate how instruction framing influences tool use and refusal behavior
- test Lumi's prompt-hardening and safety-routing logic against known prompt-shaping patterns
- build synthetic red-team scenarios for evaluation and observability

## Keep in mind

Use this resource strictly for defensive evaluation, adversarial testing, and prompt-hardening work. Do not treat it as a general training corpus or a deployment prompt source.
